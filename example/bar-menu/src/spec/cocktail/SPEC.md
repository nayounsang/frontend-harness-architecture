# 칵테일 분류 — 비즈니스 요구사항

마스터 기획은 [`../SPEC.md`](../SPEC.md)를 따른다. 공통 흐름·단일 오버레이 상세·세션 전역 상태 원칙은 마스터와 동일하다.

## 목록 화면에 맞는 정보 차원

목록 한 줄에서 손님이 비교하기 쉬운 축으로 설계한다.

| 차원 | 설명 |
|------|------|
| 이름 | 칵테일 통칭(매장 표기명). |
| 베이스 스피릿 | 진, 보드카, 럼, 테킬라, 위스키 등 주 재료 증류주. |
| 스타일 | 클래식 사워·스티어드·피즈·스매시 등 혼합 스타일 감(매장 태그로 단순화 가능). |
| 도수 감 | “낮음·중간·높음” 등 서술 단계 또는 ABV(매장 정책). |
| 서빙 글라스 | 쿠페, 올드패션드 글라스, 하이볼 등 대표 글라스. |
| 대표 재료 | 한두 개 키워드(시트러스, 허브, 비터 등). |
| 가격 | 잔·컵 단위 기준가(마스터 공통 규칙). |

품절 시 목록에 품절 표시를 병기한다(마스터 정책).

## 상세 화면 — 표시 항목

상세는 **칵테일 주종에 특화된 카피**를 제공한다. 마스터대로 **동시에 하나의 상세만** 연다.

### 필수

- 이름
- 한 줄 요약(풍미·인상)
- 베이스 스피릿 및 주요 부재료(이름 수준; 레시피 비율 공개 여부는 선택)
- 서빙 글라스·온도 감(예: 차갑게, 크러시드 아이스)
- 가격·판매 단위(잔 등)

### 선택

- 재료 목록(비율 미표기 또는 대략적 순서만)
- 알코올 도수(표기 가능 시)
- 안주 페어링 태그(예: “지방 적은 튀김과”, “과일 디저트와”) — 태그 수준으로 충분
- 매장 한줄 코멘트(시즈널 메뉴 설명 등)

품절 항목은 상세 상단에 **주문 불가**를 명확히 한다.

## 행동 규칙

- 목록에서 항목 선택 시 해당 칵테일 상세만 연다. 다른 항목 선택 시 기존 상세는 닫히고 새 상세로 교체된다.
- 분류가 칵테일일 때만 이 필드 세트를 사용한다.

## 예시 메뉴 데이터 (기획용)

아래는 스펙 검증·카피 길이 참고용 **예시**이며 실제 운영 데이터와 동일할 필요는 없다.

| 이름 | 베이스 | 스타일·특성 | 한 줄 특성 |
|------|--------|-------------|------------|
| 네그로니 | 진 | 비터·스티어드 | 진·캄파리·스위트 베르무트의 쓴맛과 허브향의 균형. 애피타이저 전에 어울림. |
| 올드 패션드 | 버번/라이 위스키 | 스티어드·숏 드링크 | 설탕과 비터로 위스키의 단맛과 향을 끌어올린 클래식. 천천히 마시기 좋음. |
| 마가리타 | 테킬라 | 사워·마가리타 글라스 | 테킬라·라임·오렌지 리큐르의 산미와 짠 림. 멕시코 요리와 잘 맞음. |
| 모히토 | 화이트 럼 | 하이볼·민트 | 민트·라임·소다의 청량감. 알코올 감은 상대적으로 부드러운 편. |
| 위스키 사워 | 위스키 | 사워·프로티 폼 옵션 | 시트러스 사워와 위스키 바디의 조합. 상큼하게 시작해 깊은 여운. |
| 진 토닉 | 진 | 하이볼 | 진의 보태니컬과 토닉의 퀴닌 쓴맛. 장시간 마시기 쉬운 표준 하이볼. |

## 마스터와의 연결

- **현재 분류**가 칵테일일 때 목록·상세는 위 차원을 사용한다.
- **열린 상세**가 칵테일 항목을 가리킬 때는 위 상세 필드를 채운다.
