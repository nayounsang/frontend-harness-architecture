# 와인 분류 — 비즈니스 요구사항

마스터 기획은 [`../SPEC.md`](../SPEC.md)를 따른다. 공통 흐름·단일 오버레이 상세·세션 전역 상태 원칙은 마스터와 동일하다.

## 목록 화면에 맞는 정보 차원

| 차원 | 설명 |
|------|------|
| 이름 | 와인명(생산자·큐베 명 등 매장 표기 규칙). |
| 타입 | 레드·화이트·로제·스파클링 등 매장 채택 분류. |
| 품종 | 카베르네 소비뇽, 피노 누아, 샤르도네 등 주 포도 품종(블렌드면 대표 품종). |
| 지역 | 국가·대표 AOC/DOC 등 고객이 기대하는 지리 라벨. |
| 바디·산도·타닌 | 와인 카드에서 흔히 쓰는 서술 축(낮음–높음 스케일 또는 태그). |
| 빈티지 | 연도; 논빈티지면 “NV” 등 표기 규칙. |
| 가격 | **글라스**와 **병** 중 매장이 파는 단위별 가격(병만일 경우 병만 표기). |

품절·비노출은 마스터 규칙을 따른다.

## 상세 화면 — 표시 항목

### 필수

- 이름·타입·품종·지역
- 빈티지 또는 NV 표기
- 테이스팅 요약: 향·입안·피니시를 **와인에 맞게** 서술(칵테일/위스키와 동일한 틀을 재사용해도 됨)
- 가격: 글라스·병 각각 매장이 제공하는 경우 모두 표시; 한쪽만이면 그에 맞춘 안내 문구

### 선택

- 추천 안주·페어링(태그 또는 한 줄)
- 권장 서빙 온도(예: “약간 시원하게”, 셀러 템프 언급)
- 디캔팅: 권장 여부를 **예/아니오/선택** 등 단순 정책으로 표시 가능
- 알코올 도수(표기 가능 시)

품절 시 해당 단위(글라스/병)별로 주문 불가를 구분해 표시할지는 매장 정책으로 정한다.

## 행동 규칙

- 목록 항목 선택 시 해당 와인 상세만 연다. 다른 항목 선택 시 단일 상세 교체(마스터).
- 분류가 와인일 때만 이 필드 세트를 사용한다.

## 예시 메뉴 데이터 (기획용)

와인 교육 자료·생산자 공개 설명과 크게 어긋나지 않는 수준의 **일반적 특성**을 예시로 둔다.

| 이름(예시) | 타입·품종·지역 | 한 줄 특성 | 목록 축 예시 |
|------------|----------------|------------|--------------|
| 보르도 좌안 블렌드 (예시 라벨) | 레드·카베르네 소비뇽 중심·프랑스 | 타닌과 구조감이 있어 스테이크류와 자주 페어링된다고 알려진 스타일. | 바디 높음·타닌 높음·산도 중간 |
| 부르고뉴 샤르도네 마을급 (예시) | 화이트·샤르도네·프랑스 | 미네랄과 산도가 강조되는 경우가 많아 해산물·크림 소스와 조합이 언급됨. | 바디 중간·산도 높음 |
| 이탈리아 키안티 클래식 (예시) | 레드·산지오베제·이탈리아 | 산도와 체리 계열 과실, 허브 노트가 특징으로 자주 소개됨. | 산도 높음·타닌 중간 |
| 스파클링 프로세코 (예시) | 스파클링·글레라·이탈리아 베네토 | 가벼운 거품과 사과·배 계열 과실, 에피타이저용으로 흔히 제안됨. | 바디 낮음–중간·산도 중간 |
| 나파 밸리 카베르네 (예시) | 레드·카베르네 소비뇽·미국 | 새 오크에서 비롯된 바닐라·블랙커런트 계열이 자주 묘사되는 뉴월드 스타일. | 바디 높음·타닌 중간–높음 |

## 마스터와의 연결

- **현재 분류**가 와인일 때 목록·상세는 위 차원을 사용한다.
- **열린 상세**가 와인 항목이면 글라스/병 가격과 서빙·페어링 정보를 채운다.
